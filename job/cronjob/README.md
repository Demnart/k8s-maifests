***Job***

Управляющая конструкция для однократного запуска приложения  
По умолчанию в случае если job завершается с ошибками, он автоматически перезапускается еще 6 раз, т.е. общее количество попыток запуска 7 раз  

Job-ы как управляющая конструкция не удаляется автоматически - как и поды запущенные с помощью Job  

Для автоматическо удаления управляющей конструкции  Job и подов созданных с помощью этой констркуции можно использовать команду - ttlSecondsAfterFinished  


***CronJob***

Запуск задач по расписания. Для указания времени используется конструкция schedule и описывается как CRON  
TIMEZONE  - можно указать конкретную временную зону в которой будет запускаться CRON
